\exercise{Hive}
\subexercise{}

The following HiveQL query was used to determine the five most frequently drawn individual lottery numbers.

\begin{verbatim}
SELECT number, COUNT(*) AS occurrences
FROM (
    SELECT explode(split(lotterynumbers,' ')) AS number
    FROM lottery
) t
GROUP BY number
ORDER BY occurrences DESC
LIMIT 5;
\end{verbatim}

The query splits the \texttt{lotterynumbers} column into individual numbers, creates one row per number using \texttt{explode}, counts the occurrences of each number, sorts the result in descending order and returns the top five numbers.

\hline

\end{tabular}

To combine multiple datasets, the taxi and fuel datasets were joined using the date attributes. The taxi dataset contains information about trip distances and earnings, while the fuel dataset contains daily fuel prices.

The following HiveQL query was used:

\begin{verbatim}
SELECT
t.dropoff_date,
SUM(t.distance) AS total_distance,
SUM(t.earnings) AS total_earnings,
f.nystate_price
FROM taxi t
JOIN fuel f
ON t.dropoff_date = f.businessday
GROUP BY t.dropoff_date, f.nystate_price
ORDER BY t.dropoff_date;
\end{verbatim}

The query joins both datasets on their corresponding dates. Afterwards, the total travelled distance and total earnings are calculated for each day and enriched with the fuel price information from the fuel dataset.

\begin{center}
\begin{tabular}{|c|c|c|c|}
\hline
Date & Total Distance & Total Earnings & NY State Fuel Price \\
\hline

2015-06-05 & 180112.00 & 1741694.44 & 2.88 \\

The first five rows of the resulting joined dataset are shown above. The results show the aggregated taxi activity for each day together with the corresponding fuel price. This demonstrates how Hive can be used to perform joins and aggregations across multiple large datasets.
